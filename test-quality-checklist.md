# FlowSpec 测试质量检查清单

## 使用说明

本检查清单用于代码审查、测试编写和质量评估。每个检查项都应该被验证，确保测试质量符合项目标准。

---

## 📋 基础测试要求

### ✅ 测试覆盖率
- [ ] 关键函数覆盖率 ≥ 90%
- [ ] 重要函数覆盖率 ≥ 85%  
- [ ] 普通函数覆盖率 ≥ 80%
- [ ] 模块整体覆盖率达到标准
- [ ] 新增代码有对应的测试

### ✅ 测试命名和结构
- [ ] 测试函数命名遵循 `Test<FunctionName>_<Scenario>` 格式
- [ ] 测试文件命名遵循 `<module>_test.go` 格式
- [ ] 使用表驱动测试处理多个测试用例
- [ ] 测试结构遵循 AAA 模式 (Arrange, Act, Assert)
- [ ] 测试用例名称清晰描述测试场景

---

## 🎯 边界条件测试

### ✅ 数据边界测试
- [ ] **空值测试**: nil, empty string, empty slice/map
- [ ] **零值测试**: 0, false, empty struct
- [ ] **最大值测试**: 最大整数、最长字符串、最大数组
- [ ] **最小值测试**: 最小整数、最短有效输入
- [ ] **临界值测试**: 边界值 ±1 的测试

### ✅ 输入验证测试
- [ ] **格式验证**: 无效 JSON、YAML、XML 格式
- [ ] **类型验证**: 错误的数据类型输入
- [ ] **范围验证**: 超出允许范围的值
- [ ] **字符集验证**: 特殊字符、Unicode、控制字符
- [ ] **长度验证**: 超长输入、空输入

### ✅ 业务逻辑边界测试
- [ ] **状态转换**: 所有有效和无效的状态转换
- [ ] **权限边界**: 权限不足、权限过期场景
- [ ] **时间边界**: 超时、时区、日期边界
- [ ] **数量边界**: 批处理大小、并发数量限制

---

## ⚠️ 错误处理测试

### ✅ 异常场景测试
- [ ] **网络异常**: 连接超时、网络中断、DNS 失败
- [ ] **文件系统异常**: 文件不存在、权限不足、磁盘满
- [ ] **内存异常**: 内存不足、内存泄漏场景
- [ ] **并发异常**: 竞争条件、死锁、资源争用

### ✅ 错误恢复测试
- [ ] **优雅降级**: 部分功能失败时的降级策略
- [ ] **重试机制**: 重试逻辑、退避策略、最大重试次数
- [ ] **回滚机制**: 操作失败时的状态回滚
- [ ] **错误传播**: 错误信息的正确传播和包装

### ✅ 错误信息质量
- [ ] **错误信息清晰**: 用户可理解的错误描述
- [ ] **错误分类正确**: 错误类型和严重级别正确
- [ ] **调试信息充分**: 包含足够的上下文信息
- [ ] **国际化支持**: 多语言错误信息

---

## 🔄 并发安全测试

### ✅ 数据竞争测试
- [ ] **共享状态访问**: 多线程访问共享变量
- [ ] **缓存一致性**: 缓存更新和读取的一致性
- [ ] **状态同步**: 状态变更的同步机制

### ✅ 资源管理测试
- [ ] **连接池管理**: 连接获取、释放、超时
- [ ] **内存管理**: 并发内存分配和释放
- [ ] **文件句柄管理**: 并发文件操作

### ✅ 性能测试
- [ ] **并发性能**: 高并发下的性能表现
- [ ] **资源使用**: CPU、内存使用情况
- [ ] **响应时间**: 并发场景下的响应时间

---

## 📊 测试质量和稳定性

### ✅ 测试稳定性
- [ ] **可重现性**: 测试结果一致且可重现
- [ ] **环境独立性**: 测试不依赖特定环境
- [ ] **数据隔离**: 测试数据相互隔离
- [ ] **时间独立性**: 避免时间相关的不稳定测试

### ✅ 测试性能
- [ ] **执行时间**: 单元测试 < 30秒，集成测试 < 2分钟
- [ ] **资源使用**: 测试不消耗过多内存和 CPU
- [ ] **并行友好**: 测试可以并行执行
- [ ] **快速反馈**: 测试失败时提供快速反馈

### ✅ 测试维护性
- [ ] **代码清晰**: 测试代码易读易理解
- [ ] **注释充分**: 复杂测试逻辑有适当注释
- [ ] **重复最小**: 避免测试代码重复
- [ ] **易于修改**: 业务逻辑变更时测试易于更新

## 📝 使用示例

### 代码审查时使用
```markdown
## 测试质量检查

- [x] 测试覆盖率达标 (新增函数覆盖率 95%)
- [x] 边界条件测试充分 (包含空值、最大值、异常输入测试)
- [x] 错误处理测试完整 (网络异常、文件错误、超时场景)
- [ ] 并发安全测试需要补充 (缺少并发访问测试)
- [x] 测试命名和结构符合规范

**需要改进**: 请添加并发访问的测试用例，特别是多线程同时调用 `processData` 函数的场景。
```

### 开发自检时使用
```markdown
## 开发自检清单

功能: 添加新的数据验证逻辑

- [x] 基础功能测试 (正常输入、有效数据)
- [x] 边界条件测试 (空值、最大值、最小值)
- [x] 错误处理测试 (无效格式、类型错误)
- [x] 性能测试 (大数据量处理)
- [x] 测试命名规范 (TestValidateData_*)
- [x] 本地测试全部通过

准备提交代码审查。
```
